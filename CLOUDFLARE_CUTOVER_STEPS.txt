Cloudflare changes required

1. Log into Cloudflare
- Open the Cloudflare dashboard.
- Select the zone for:
  - webelieveacademics.com

2. Check SSL/TLS mode first
- Go to:
  - SSL/TLS
- Find the current encryption mode.
- Set it to:
  - Full
- Do not leave it on Full (strict) until the origin gets a proper trusted cert.

3. Open DNS settings
- Go to:
  - DNS

4. Update the apex/root record
- Find the DNS record for:
  - webelieveacademics.com
- It will normally be one of:
  - A record for @
  - flattened CNAME for @
- Change the origin target so that traffic ultimately points to:
  - 108.249.142.225

Recommended target:
- Type: A
- Name: @
- IPv4 address: 108.249.142.225
- Proxy status: Proxied

5. Update the www record
- Find the record for:
  - www
- Point it to the same origin.

Recommended options:
- Option A:
  - Type: CNAME
  - Name: www
  - Target: webelieveacademics.com
  - Proxy status: Proxied

- Option B:
  - Type: A
  - Name: www
  - IPv4 address: 108.249.142.225
  - Proxy status: Proxied

Preferred approach:
- Use CNAME for www -> webelieveacademics.com if convenient.

6. Save the DNS changes
- Confirm the edited records are saved.

7. Optional: lower TTL if Cloudflare allows manual TTL on any DNS-only records
- If TTL is editable and you want faster recheck behavior, lower it before finalizing.
- If records are Proxied, Cloudflare may abstract TTL behavior.

8. Do NOT change unrelated records
- Only update:
  - webelieveacademics.com
  - www.webelieveacademics.com
- Do not alter mail, verification, or other subdomain records unless you intend to move them too.
